<p>We’re excited to announce a new set of pre-written Sentinel policies for AWS designed to support ISO/IEC 27001 compliance. These policy sets aim to lower the barrier to adopting policy as code while helping organizations align their Terraform-managed AWS resources with globally recognized security standards. Co-created and co-owned by HashiCorp and AWS, these policies map to key ISO 27001 Annex A controls and are now available for use in the Terraform Registry. </p><h3>A collaborative and continued effort </h3><p>While Sentinel is a powerful tool for enforcing cloud governance at scale, adopting a policy-as-code model can be complex and resource-intensive. Many organizations face challenges due to limited expertise or the time required to translate compliance frameworks into enforceable policies from scratch. </p><p>To help address these challenges, AWS and HashiCorp have worked closely to simplify hybrid-cloud governance and compliance. Building on previous releases aligned to frameworks such as CIS and AWS Foundational Security Best Practices (FSBP), we are excited to introduce a new set of pre-written Sentinel policies designed to help organizations meet ISO/IEC 27001 requirements across AWS environments. These policies map to key ISO 27001 Annex A controls—covering areas such as access control, cryptography, logging and monitoring, and secure configuration management. </p><p>With hundreds of policies now available, organizations can adopt a secure-by-default posture for AWS infrastructure without having to start from scratch. These policy sets provide a strong foundation, reducing the need for manual policy development and helping teams consistently enforce governance controls across their cloud environments. </p><h3>Getting started with pre-written Sentinel policies </h3><p>Please refer to our <a href="https://developer.hashicorp.com/terraform/cloud-docs/workspaces/policy-enforcement/prewritten-library#iso-iec-27001-2013-annex-a" rel="noreferrer noopener" target="_blank"><u>pre-written policy library documentation</u></a> to get started with pre-written Sentinel policies for ISO 27001, or check out the other pre-written Sentinel policy sets available in the Terraform registry: </p><ul><li><p><a href="https://registry.terraform.io/policies/hashicorp/CIS-Policy-Set-for-AWS-Terraform/1.0.2" rel="noreferrer noopener" target="_blank"><u>Pre-written Sentinel Policies for AWS CIS Foundations Benchmarking</u></a> </p></li><li><p><a href="https://registry.terraform.io/policies/hashicorp/FSBP-Policy-Set-for-AWS-Terraform/1.1.2" rel="noreferrer noopener" target="_blank"><u>Pre-written Sentinel Policies for AWS Foundational Security Best Practices (FSBP)</u></a> </p></li><li><p><a href="https://registry.terraform.io/policies/hashicorp/nist-policy-set-for-aws-terraform/1.0.1" rel="noreferrer noopener" target="_blank"><u>Pre-written Sentinel Policies for AWS NIST SP 800-53 Revision 5</u></a> </p></li><li><p><a href="https://registry.terraform.io/policies/hashicorp/pcidss-policy-set-for-aws-terraform/1.0.1" rel="noreferrer noopener" target="_blank"><u>Pre-written Sentinel Policies for AWS PCI DSS</u></a> </p></li><li><p><a href="https://registry.terraform.io/policies/hashicorp/aws-networking-terraform/1.0.1" rel="noreferrer noopener" target="_blank"><u>AWS Networking Sentinel Policies for Terraform</u></a> </p></li></ul><p> </p><p>New to Terraform?
